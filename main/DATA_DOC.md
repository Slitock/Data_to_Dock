# denormalizedFeeds
## Описание

denormalizedFeeds - это метаданные таблицы, которая хранит информацию о товарах в распределенном формате. Это необработанная версия данных, которая будет использоваться для создания более сложных представлений и аналитики.

## Логика расчета

В этом разделе описывается бизнес-логика объединения данных. Данные таблицы - это частично нормализованные данные о товарах, которые были получены из различных источников (например, из кэша и других систем). Данные были объединены для создания более полного представления о товарах.

В частности, была сделана следующая логика:
* sku_id - это уникальный идентификатор товара.
* qty_rem - это количество оставшихся единиц товара.
* base_price - это базовая цена товара.
* category - это категория товара.
* last_upd - это дата последнего обновления информации о товаре.
* storage_zone - это зона хранения товара.
* is_active - это флаг, указывающий, является ли товар активным или неактивным.

## Источники данных

Данные хранятся в распределенном формате на HDFS/Hive. В частности, файл bd.csv содержит данные о товарах.

Кроме того, были использованы следующие топики Kafka:
* ok.feed.update - для обновлений информации о товаре.
* ok.feed.show - для событий "показа" товара.
* ok.feed.click - для событий "клика" по товарам.

## Описание данных (Схема)

### Столбцы

| Название | Тип | Описание | Примеры значений |
| --- | --- | --- | --- |
| sku_id | str | Уникальный идентификатор товара | A-105, B-202, C-303 |
| qty_rem | float64 | Количество оставшихся единиц товара | 140.0, 12.0, 0.0 |

## Детализация полей

### sku_id

Тип поля - string. Описание: уникальный идентификатор товара.

Пример парсинга:
```sql
SELECT SKU_ID FROM denormalizedFeeds WHERE SKU_ID = 'A-105'
```

### last_upd

Тип поля - string. Описание: дата последнего обновления информации о товаре, в формате YYYY-MM-DD.

Пример парсинга:
```sql
SELECT LAST_UPD FROM denormalizedFeeds WHERE LAST_UPD = '2026-02-08'
```

### is_active

Тип поля - object. Описание: флаг, указывающий, является ли товар активным или неактивным.

Вложенность:
* active: boolean
* reason: string (optional)

Пример парсинга:
```sql
SELECT IS_ACTIVE FROM denormalizedFeeds WHERE IS_ACTIVE = true
```

## Известные проблемы и Инциденты

### 1. Баг логирования времени на Android

В ходе тестирования было обнаружено, что в некоторых случаях система не могла правильно записать дату и время событий.

### 2. Пропуски при джойне по renderId

Было обнаружено, что в редких случаях произошло пропуск события "клика" при джойне по renderId. Восстановление было успешно выполнено после отладки и исправления ошибки.

### 3. Ошибка вычислений qty_rem

В ходе тестирования была обнаружена ошибка вычислений в поле qty_rem, которая привела к неправильному результату. Ошибка была исправлена, и данные были обновлены.
